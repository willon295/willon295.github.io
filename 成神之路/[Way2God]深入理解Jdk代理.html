# [Way2God]深入理解Jdk代理

# 1. 名词解释

  1. java.lang.reflect.Proxy : 代理工厂类 , 负责生成 代理对象
  2. java.lang.reflect.InvocationHandler: Handler接口. 它的 实现类 , 用于 拦截 (非真正意义)被代理的方法, 改为调用 handle 实现类 的 invoke 方法. 且invoke方法会拥有 被代理方法 的 所有信息 .

本节只是做简单名词解释, 可能和网上常见的说法有差异, 我们暂且按这个思路走一遍

# 2. 先看一个DEMO

# 2.1 Hello接口

需要被代理的接口及方法

public interface Hello {
    String say(String word);
}

# 2.2 HelloHandler

该类实现 java.lang.reflect.InvocationHandler , 且在方法 invoke 中打印方法参数

public class HelloHandler implements InvocationHandler {
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("==========================");
        System.out.println("HelloHandler invoke");
        System.out.println("proxy: " + proxy.getClass().getCanonicalName());
        System.out.println("method: " + method.getDeclaringClass().getCanonicalName()+"#"+method.getName());
        System.out.println("args: " + Arrays.toString(args));
        System.out.println("==========================");
        return "Hello" + Arrays.toString(args);
    }
}

# 2.3 测试


public static void main(String[] args) throws IOException {
        // 1. 创建一个handler
        HelloHandler handler = new HelloHandler();
        
        // 2. 创建代理对象
        Object proxyInstance = Proxy.newProxyInstance(Hello.class.getClassLoader(), new Class[]{Hello.class}, handler);
        
        // 3. 强转成 Hello
        Hello hello = (Hello) proxyInstance;
        System.out.println("proxyInstanceClassName: "+hello.getClass().getCanonicalName());

        // 4. 调用 Hello 的 say 方法
        String word = hello.say("word");
        System.out.println("return: "+word);
}

查看输出:

==========================
HelloHandler invoke
proxy: com.sun.proxy.$Proxy0
method: cn.willon.proxy.Hello#say

# 2.4 测试结论

  1. Proxy生成的 代理实例 的全类名为 : com.sun.proxy.$Proxy0
  2. 代理实例 com.sun.proxy.$Proxy0可以被强转成 Hello
  3. handler 的 invoke 方法被执行, 且拥有所有 Hello.say() 方法的信息

# 3. 底层原理?

那么 com.sun.proxy.$Proxy0 到底是什么东西, 我如此杠精, 自然是要刨根问底的.

再对 Proxy.newProxyInstance() 方法进行追踪之后, 发现最终创建代理类的关键代码如下

/*
* Generate the specified proxy class.
*/
// 这里就是生成 代理类的关键字节码信息
byte[] proxyClassFile = ProxyGenerator.generateProxyClass(
                proxyName, interfaces, accessFlags);
// 定义一个代理类Class
return defineClass0(loader, proxyName, proxyClassFile, 0, proxyClassFile.length);

那么为了观察这个生成的代理对象, 我们需要继续观察: sun.misc.ProxyGenerator#generateProxyClass(java.lang.String, java.lang.Class<?>[], int) 这个方法

我们看到有一个 saveGeneratedFiles 的 属性, 它的定义如下

private final static boolean saveGeneratedFiles =
        java.security.AccessController.doPrivileged(
            new GetBooleanAction( "sun.misc.ProxyGenerator.saveGeneratedFiles")).booleanValue();

从定义得知, 我们只需要将 系统参数 sun.misc.ProxyGenerator.saveGeneratedFiles 设置为 true 就可以得到生成的文件, (事实上还要多一步)

# 3.1 准备工作:获取代理对象class

现在我们对测试入口 main 方法进行修改,

  1. 在 原有创建代理对象的
Object proxyInstance = Proxy.newProxyInstance(Hello.class.getClassLoader(), new Class[]{Hello.class}, handler);

语句之前加上:

System.getProperties().put("sun.misc.ProxyGenerator.saveGeneratedFiles", "true");

含义是: 保存代理生成的 class文件

  1. 在工程的 根目录 , 手动创建文件夹 com/sun/proxy/ , 在文件夹内新建空文件 $Proxy0.class
  2. 再次运行 main 函数, 此时 $Proxy0.class 已经有内容

# 4.思考

本文大致介绍了 JDK 如何代理一个 Hello 接口的 say 方法 , 代理实例对象的方法, 原理也是如此.前提是 被代理 的类必须实现了某些接口 .

# 4.1 解答疑惑

Q1: 为什么jdk代理创建代理对象时, 只能传入 接口 ?
A1: 底层是生成 $Proxy0 类, 该类默认 继承 Proxy, 根据Java不可多继承原则, 只能传入接口
Q2: 为什么Mabatis只需要定义接口和MapperXml就能实现接口调用
A1: 从jdk代理的原理看来, 可以生成一个代理对象, 实现该接口, 进而实现接口方法调用

# 4.2 什么导致Java实现代理如此复杂

本人认为有两点

  1. Java不支持多继承
  2. Java不支持动态继承,动态实现. (Scala语言支持), 以下为Scala创建实例时动态实现一个接口代码
var b = new Bird with Flyable {
   override def fly(): String = {
        "bird fly"
   }
}
  1. 原生JDK生成字节码的API并不对外暴露, 导致想动态定义一个Class变得复杂 , 这一点可以通过第三方框架解决, 如 ASM,CGLIB
