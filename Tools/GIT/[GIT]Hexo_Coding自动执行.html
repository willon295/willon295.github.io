痛苦来源

每次写文章要做那么多事

主要功能

本地写.md文章之后，执行post.sh ，自动完成

  • 生成静态页面
  • 推送并更新github page（或者coding page）
  • 推送博客文章源码到远程仓库

动机

动机当然是解决那些痛苦了

谈谈github+TravisC

本人认为最好的Hexo自动部署方案就是 github+TravisCI

  1. githu+TravisCI优点

    1. 无需安装Nodejs
    2. 无需安装Hexo
    3. 只需管理文章源文件

  2. githu+TravisCI不足

    1. 无法本地灵活调试Hexo
    2. TravisCI容易部署失败，原因各种各样，文章的一个标点都会让一次部署失败，排错较难
    3. github page已屏蔽百度spider,所以不利于网站收录
    4. 访问速度没国内coding快

于是决定想个办法实现coding自动部署Hexo。

  1. 安装Hexo,Nodejs
